# あきらパパ自己分析会議（最小）

## Description
複数キャラクターで「自己分析会議」を行い、**1つの議事録ノート**にまとめます（フォルダ増殖を避ける）。

- 出力先（おすすめ）: `Atlas/YYYY-MM-DD-あきらパパ自己分析会議.md`

## Prompt
次を守って自己分析会議を実施してください。

### 1) 参照する情報（最小）
必ず読んでください：
- `Atlas/00-note戦略.md`（発信の軸）
- `Atlas/文章スタイルガイド.md`（文章の型）

余裕があれば確認：
- `Calendar/inbox/` の直近メモ（最新5〜10件）
- `Efforts/*/00-概要.md`（`archived/` は除外。今の活動/制約を拾う）

### 2) 参加キャラクター（例）
次の4視点で会議形式にしてください（必要なら増やしてOK）：
- エンジニア視点（技術/実装/ツール）
- 教育者視点（教える/設計/再現性）
- ビジネス視点（価値提供/収益/戦略）
- 家族・健康視点（持続性/生活/体力）

### 3) 出力ファイル
新規に `Atlas/YYYY-MM-DD-あきらパパ自己分析会議.md` を作成し、以下の形式で出力してください。

```markdown
---
title: YYYY-MM-DD-あきらパパ自己分析会議
tags: [personal, review]
created: YYYY-MM-DD
---

# YYYY-MM-DD-あきらパパ自己分析会議

- ...

## 参加者
- エンジニア:
- 教育者:
- ビジネス:
- 家族・健康:

## 議事録
### 1. 現状（事実）
- ...

### 2. 課題（論点）
- ...

### 3. 仮説（なぜ起きているか）
- ...

### 4. 次の一手（Top5）
1. ...
2. ...
3. ...
4. ...
5. ...

## 関連
- `Atlas/_Personal-MOC.md` へのリンク候補（あれば）
- `Efforts/*/00-概要.md` の更新が必要なら指摘
```

### 4) 仕上げ（最小）
- タイトル・タグを最小で付ける（過剰に増やさない）
- `Atlas/🗺️ Atlas-MOC.md` か `Atlas/_Personal-MOC.md` にリンクを1つだけ追加提案
