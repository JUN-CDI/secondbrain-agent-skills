# 知識調査コマンド

## Description
指定テーマを網羅的に調査し、**細分化したノート**を複数作成します。
作成先は `Calendar/inbox/research/`（後で整理して `Atlas/` へ）。

## Prompt
以下のテーマについて、網羅的に詳細な知識を調査して、`Calendar/inbox/research/` に複数のMarkdownファイルを作成してください。

**重要**: 1項目1ファイルの原則を厳守してください（機能単位・概念単位で必ず別ファイル）。

引数:
- テーマ: 調査したい技術や概念（例: "React Hooks", "TypeScript", "PostgreSQL"）

要件:
1. 調査項目を洗い出す（基本概念/主要機能/実装/ベストプラクティス/よくある問題/性能/セキュリティ/比較）
2. テーマを細分化し、最低5ファイル以上（通常10〜20）を作る計画を立てる
3. 各サブトピックを **1ファイルで完結**する形で執筆（コード例・使用例を含める）
4. 作成先: `Calendar/inbox/research/`
   - ファイル名: `YYYY-MM-DD-[テーマ]-[サブトピック].md`（日本語推奨、具体的に）
5. 可能な範囲で相互リンクを追加
6. 最後に「作成一覧＋次の整理案（Memory候補など）」をレポートする

補足:
- 週次で `/organize-input` を回して `Atlas/` に昇華する運用を想定
