# 🎯 Cursorレイアウト設定 - 実践手順

**目的:** Cursorの画面で実際に操作して、レイアウトを設定する

**所要時間:** 約5分

---

## 📋 全体プラン

1. ✅ **Step 1:** 左側Explorerを表示・固定
2. ✅ **Step 2:** ターミナルパネルを右側に移動
3. ✅ **Step 3:** ターミナルを分割（claude/codex）
4. ✅ **Step 4:** AIチャットの設定確認
5. ✅ **Step 5:** tasks.jsonの動作確認（オプション）

---

## ✅ Step 1: 左側Explorerを表示・固定

### 目的
ファイルツリーを常時表示する

### 操作手順

1. **サイドバーを表示**
   - キーボード: `Cmd+B` (macOS) / `Ctrl+B` (Windows/Linux)
   - または: メニュー → `View` → `Appearance` → `Show Primary Side Bar`

2. **Explorerを選択**
   - 左側のサイドバーで、一番上の **📁 Explorer** アイコンをクリック
   - または: `Cmd+Shift+E` (macOS) / `Ctrl+Shift+E` (Windows/Linux)

3. **サイドバーを固定（ピン留め）**
   - サイドバーが表示されている状態で、サイドバーの右端（エディタとの境界）を確認
   - サイドバーが自動的に固定されているはずです
   - もし閉じてしまう場合は、`Cmd+B` で再度表示

### 確認ポイント
- ✅ 左側にファイルツリーが表示されている
- ✅ `Cmd+B` で閉じても、再度 `Cmd+B` で開ける

### 完了したら
→ **Step 2** に進む

---

## ✅ Step 2: ターミナルパネルを右側に移動

### 目的
ターミナルを縦方向に配置して、コードエリアを広く使う

### 操作手順

#### 方法A: ドラッグ＆ドロップ（推奨）

1. **ターミナルパネルを開く**
   - キーボード: `Cmd+Shift+\`` (macOS) / `Ctrl+Shift+\`` (Windows/Linux)
   - または: メニュー → `View` → `Terminal`
   - または: メニュー → `Terminal` → `New Terminal`

2. **ターミナルパネルのタイトルバーを確認**
   - ターミナルが開いたら、上部に「TERMINAL」というタイトルバーがあるはずです

3. **タイトルバーをドラッグ**
   - タイトルバーを**マウスでクリック＆ドラッグ**
   - 画面の**右端**までドラッグ
   - 右端に近づくと、ドロップゾーンが表示されるはずです

4. **右端にドロップ**
   - 右端のドロップゾーンにドロップ
   - ターミナルが縦方向のパネルとして配置されます

#### 方法B: コマンドパレット（方法Aがうまくいかない場合）

1. **コマンドパレットを開く**
   - `Cmd+Shift+P` (macOS) / `Ctrl+Shift+P` (Windows/Linux)

2. **「Move Panel Right」を検索**
   - `Move Panel Right` と入力
   - `View: Move Panel Right` を選択

### 確認ポイント
- ✅ ターミナルが画面右側に縦に表示されている
- ✅ エディタエリアが中央に残っている

### トラブルシューティング
- **ターミナルが右に移動しない場合:**
  1. ターミナルパネルを一旦閉じる（`Cmd+J` または `Cmd+Shift+\``）
  2. メニュー → `View` → `Appearance` → `Panel Position` → `Right` を選択
  3. ターミナルを再度開く（`Cmd+Shift+\``）

### 完了したら
→ **Step 3** に進む

---

## ✅ Step 3: ターミナルを分割（claude/codex）

### 目的
2つのAIエージェント用に別々のターミナルを用意

### 操作手順

1. **最初のターミナルを確認**
   - 右側のターミナルパネルに、1つのターミナルが表示されているはずです
   - タブに「1: bash」や「1: zsh」などと表示されている

2. **ターミナルタブをリネーム（claude）**
   - ターミナルタブを**右クリック**
   - `Rename` を選択
   - `claude` と入力して Enter

3. **ターミナルを分割**
   - キーボード: `Cmd+\` (macOS) / `Ctrl+\` (Windows/Linux)
   - または: ターミナルタブを右クリック → `Split Terminal`
   - または: ターミナルパネル上部の分割ボタン（⚡アイコン）をクリック

4. **2つ目のターミナルタブをリネーム（codex）**
   - 新しく作成されたターミナルタブを**右クリック**
   - `Rename` を選択
   - `codex` と入力して Enter

### 確認ポイント
- ✅ 右側パネルに `claude` と `codex` の2つのターミナルが表示されている
- ✅ 2つのターミナルが縦に分割されている

### トラブルシューティング
- **分割がうまくいかない場合:**
  1. 既存のターミナルを全て閉じる（タブを右クリック → `Kill Terminal`）
  2. 新規ターミナルを開く（`Cmd+Shift+\``）
  3. リネーム → `claude`
  4. 分割（`Cmd+\`）
  5. リネーム → `codex`

### 完了したら
→ **Step 4** に進む

---

## ✅ Step 4: AIチャットの設定確認

### 目的
AIチャットはデフォルトで閉じて、必要な時だけ開く設定にする

### 操作手順

1. **AIチャットを閉じる（開いている場合）**
   - キーボード: `Cmd+Shift+A` (macOS) / `Ctrl+Shift+A` (Windows/Linux)
   - または: 右側のAIチャットパネルを閉じる（×ボタン）

2. **動作確認**
   - `Cmd+Shift+A` で開く
   - `Cmd+Shift+A` で閉じる
   - この動作がスムーズにできることを確認

### 確認ポイント
- ✅ AIチャットが閉じた状態で作業できる
- ✅ `Cmd+Shift+A` で開閉できる

### ベストプラクティス
- **通常作業時:** AIチャットは閉じておく
- **プロンプト入力時:** `Cmd+Shift+A` で開いて使用
- **終わったら:** すぐに閉じる

### 完了したら
→ **Step 5** に進む（オプション）

---

## ✅ Step 5: tasks.jsonの動作確認（オプション）

### 目的
自動セットアップが動作するか確認する

### 前提条件
- `.vscode/tasks.json` が既に作成されていること

### 操作手順

1. **コマンドパレットを開く**
   - `Cmd+Shift+P` (macOS) / `Ctrl+Shift+P` (Windows/Linux)

2. **「Run Task」を検索**
   - `Run Task` と入力
   - `Tasks: Run Task` を選択

3. **「setup-ai-terminals」を選択**
   - タスク一覧から `setup-ai-terminals` を選択
   - または、`claude-terminal` と `codex-terminal` を個別に実行

4. **動作確認**
   - claude と codex のターミナルが自動的に起動するか確認
   - 分割表示されるか確認
   - **重要**: 実際に `claude` と `codex` コマンドが実行され、対話型セッションが開始されます

### 確認ポイント
- ✅ タスクが正常に実行される
- ✅ 2つのターミナルが自動的に起動する
- ✅ 分割表示される
- ✅ claudeターミナルでClaude Codeが起動する
- ✅ codexターミナルでCodexが起動する

### トラブルシューティング
- **タスクが見つからない場合:**
  1. `.vscode/tasks.json` のパスを確認
  2. Cursorを再起動
  3. コマンドパレットから再度実行

- **ターミナルが起動しない場合:**
  1. 手動でStep 3を実行してターミナルを作成
  2. tasks.jsonは補助的なものとして使用

- **claude/codexコマンドが見つからない場合:**
  1. `which claude` と `which codex` でパスを確認
  2. パスが通っていない場合は、フルパスを指定（例: `/opt/homebrew/bin/claude`）
  3. インストールされていない場合は、それぞれのCLIをインストール

### 完了したら
→ **🎉 設定完了！**
