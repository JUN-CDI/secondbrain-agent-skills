# 知識調査プロンプト

## 使用タイミング
指定テーマについて、網羅的に知識を調査したい時

## プロンプト本文
以下のテーマについて、網羅的に詳細な知識を調査して、`Calendar/inbox/research/` に複数のMarkdownファイルを作成してください。

**重要**: 1項目1ファイルの原則を厳守してください（機能単位・概念単位で必ず別ファイル）。

引数:
- テーマ: 調査したい技術や概念（例: "React Hooks", "TypeScript", "PostgreSQL"）

要件:
1. 調査項目を洗い出す（基本/主要機能/実装/ベストプラクティス/問題/性能/セキュリティ/比較）
2. 1トピック=1ファイルで、最低5ファイル以上（通常10〜20）作る
3. 各ファイルは `YYYY-MM-DD-[テーマ]-[サブトピック].md` の形式
4. 作成先は `Calendar/inbox/research/`
5. 最後に作成一覧と、`Atlas` に昇華すべき候補をレポートする

## 使用例
- `/research React Hooks`

## 期待される結果
- 複数のMarkdownファイルが `Calendar/inbox/research/` に作成される
- 後で週次に `/organize-input` で `Atlas/` へ移動できる
