# 引き継ぎ-開発作業（Codex/Claude Code）

## 目的
作業の「現状」と「判断材料」を、次の担当（Codex/Claude Code）に短く正確に渡す。

## 再開SSOT（最重要）
- **repo直下の `handoff.md`** が唯一の再開SSOT
  - ツール切替/不安時: `handoff quick`（Nextだけ入力）
  - その日の終わり: `wrapup`（推奨: `git add -p` → `endwork`）/ `endwork`（最小: `handoff full` + `handoff.md` のみcommit）

## Claude Code Skill
- `dev-handoff`（「dev-handoff」「作業状況まとめ」「開発引き継ぎサマリ」で発動）

## 最低限のチェック（5分）
0. `handoff.md` を確認（必要なら `handoff quick` で更新）
   - `docs/HANDOFF.md` は任意（長文の補助）
1. `git status -sb`
2. `git diff`（未ステージ）
3. `git add -p`（レビュー対象だけをステージ）
4. `git diff --staged`（レビュー対象の確定）
5. 実行した確認（テスト/手動検証）をメモ

## 引き継ぎテンプレ（そのまま貼る）
```markdown
# 引き継ぎサマリ

## 目的
- 何を達成する変更か（1〜2行）

## Repo / Branch
- Repo: <path>
- Branch: <branch-name>

## 変更の範囲
- ステージ済み: <files / 概要>
- 未ステージ: <files / 概要>

## Handoffファイル
- `handoff.md` を更新済み（Nextがそのまま実行できる状態）
- `docs/HANDOFF.md` は任意（必要なら要点を一行で）

## 重要な判断/合意
- <決定事項>
- <除外事項/運用ルール>

## 実行した確認
- <テスト/手動確認/結果>

## 次にやること
- <次の作業>

## 不明点/質問
- <確認したいこと>
```

## Codexに渡すときの最小セット
以下の3点を貼れば、レビュー/引き継ぎが進む:
- `git status -sb`
- `git diff --staged --stat`
- 上の「引き継ぎサマリ」
