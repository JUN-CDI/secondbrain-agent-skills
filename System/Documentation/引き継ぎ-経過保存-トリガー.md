# 引き継ぎ / 経過保存 - トリガー（最小）

目的: ツールを跨いでも「どこまでやったか / 次に何をするか」を迷わず復元できる状態にする。

## まず最小（迷ったらこれだけ）

次のどれかに当てはまったら、対象repoで **`handoff quick`** を実行して `handoff.md` を更新する。

- ツール/作業場所を切り替える前（Cursor ↔ Claude Code ↔ Codex / Vault ↔ Repo）
- 30分以上中断する前
- コミット/PRなど区切りを付けた直後
- 「次の一手」が1つに固まった時

## 何を更新する？（1分で済ませる）

### Repo（再開SSOT）
- **`handoff.md`（repo直下）** が唯一の再開SSOT
- `handoff quick` は **Next（次の最初の1アクション）だけ**入力すればOK（他は自動補完/carry）

```bash
cd ~/Workspaces/projects/<repo>
handoff quick --next "cd backend && pytest -q"
```

### Repo（開発の引き継ぎ）
- `docs/HANDOFF.md`（任意・長文OKの補助ドキュメント）
  - 完成版の要点/使い方/背景（必要なときだけ）

### Git（経過保存の最小）
- 迷ったら: `git add -p` → `git diff --staged` → commit（コードの経過保存）
- セッション終了時は `wrapup`（推奨: `git add -p` → `endwork`）/ `endwork`（最小: `handoff.md` のみcommit、他を巻き込まない）

## トリガー別の推奨アクション

- **ツール切替/不安**: `handoff quick`（Nextだけ入れて進捗を固定）
- **実装の区切り**: commit（小さく）＋ `handoff quick`
- **作業終了**: `wrapup`（推奨）/ `endwork`（最小）
- **レビュー依頼**: `git add -p` でステージ → `diff-review`（またはCodexレビュー）→ 反映
- **データ混入が不安**: `data-safety`（samples/deliverables/CSV/XLSXの追跡チェック）
