# 引き継ぎ / 経過保存 - トリガー（最小）

目的: ツールを跨いでも「どこまでやったか / 次に何をするか」を迷わず復元できる状態にする。

## まず最小（迷ったらこれだけ）

次のどれかに当てはまったら、`System/Documentation/引き継ぎ.md` に1行追記。

- Cursor: `/handoff`
- それ以外（Codex拡張/CLI、外部ターミナル等）: `./System/Scripts/handoff-add.sh "<場所 or ファイル>" "<一言>"`

- ツール/作業場所を切り替える前（Cursor ↔ Claude Code ↔ Codex / Vault ↔ Repo）
- 30分以上中断する前
- コミット/PRなど区切りを付けた直後
- 「次の一手」が1つに固まった時

## 何を更新する？（1分で済ませる）

### Vault（共通の作業状態）
- `System/Documentation/引き継ぎ.md`
  - 形式: `- HH:MM <変更ファイルパス> - <一言>`
  - 変更が多い場合は3〜10件に要約

補足（ツール非依存の追記）:
```bash
cd ~/Library/Mobile\ Documents/iCloud~md~obsidian/Documents/SecondBrain
./System/Scripts/handoff-add.sh "~/Workspaces/projects/<repo>" "Codexレビュー完了（指摘はdocs/HANDOFF.mdへ）"
```

### Repo（開発の引き継ぎ）
- `docs/HANDOFF.md`（ある場合）
  - 目的/差分/実行した確認/次の作業

### Git（経過保存の最小）
- 迷ったら: `git add -p` → `git diff --staged` → commit
  - テスト/捨て作業なら commit せず `引き継ぎ.md` に「次の一手」だけ残す

## トリガー別の推奨アクション

- **実装の区切り**: commit（小さく）＋ `引き継ぎ.md` に一行
- **レビュー依頼**: `git add -p` でステージ → `diff-review`（またはCodexレビュー）→ 反映
- **データ混入が不安**: `data-safety`（samples/deliverables/CSV/XLSXの追跡チェック）
